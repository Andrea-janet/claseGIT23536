Clase GIT/gitHub

Primeros pasos para trabajar con GIT y GitHub

Trabajando con GIT (repositorio LOCAL)

  1. Crear un directorio (una carpeta contenedora del proyecto)
  2. Abrir el directorio con Visual Studio Code (VSC)
  3. En VSC ir a TERMINAL > Nueva Terminal. Podemos elegir en "+" la opcion de Terminal Bash
  4. En la terminal, Inicializamos el repositorio con el comando git init, que se usa por unica vez en este proyecto.
  5. Ahora agregamos contenido al directorio (archivos, otras carpetas) en el Editor de VSC
  6. Podemos ver el estado de nuestro repositorio usando un comando: git status. En este punto nos mostrará que nuestro contenido no tiene seguimiento, eso quiere decir que no está en la etapa de staging. Nos damos cuenta porque los nombres de archivos y carpetas se enlistan en rojo.
  7. Para agregar el contenido del directorio a la etapa de staging tenemos que usar el comando git add "nombre_de_tu_archivo.ext" o git add . . El último de estos comandos agrega todos los archivos y carpetas automáticamente. Ahora si volvemos a usar "git status", los archivos y carpetas se enlistarán en verde.
  8. Ahora ya podemos enviar todo el directorio del área de staging al Repositorio LOCAL usando el comando git commit -m "agrega un mensaje descriptivo aca". A ésto se lo conoce como hacer commit.

Trabajando con GitHub(repositorio REMOTO)

Teniendo todo el directorio en el repositorio Local, ya podemos trabajar con la página GitHub, que es un repositorio remoto.

  1. Habiendo ingresado a nuestra cuenta, primero creamos un nuevo repositorio remoto (boton verde) al que le vamos a dar un nombre y presionaremos continuar
  2. A él enviaremos desde la Terminal Bash de VSC todo el repositorio LOCAL.

GitHub nos proporcionará todos los comandos necesarios para trabajar en la terminal de VSC.

Trabajando con GIT y GitHub en conjunto

Copiamos uno por uno los comandos proporcionados por git, en el orden que son dados.

  1. Copiamos el comando git branch -M main de GitHub y lo pegamos en la terminal VSC
  2. Seguimos con git remote add origin https://github.com/TuUsuario/TuRepositorio.git, el cual nos permite vincular ambos repositorios
  3. Finalizamos con git push -u origin main, que envía todo lo que está en GIT (R. local) a GitHub (R. remoto). La página de GH nos va a pedir una autenticación en el navegador, damos a aceptar.
