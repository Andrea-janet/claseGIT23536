Clase GIT/gitHub

Primeros pasos para trabajar con GIT y GitHub

Para realizar estos pasos ya tenemos que tener instalado GIT en nuestro dispositivo.

Creando un usuario en GIT y GitHub

Es indistínto cuál usuario creemos primero, pero para trabajar con ambos repositorios es necesario tenerlos.

  • Creando un usuario en GIT

    Para Registrarnos en GIT usamos dos comandos, este paso es escencial pero se realiza por única vez, no debemos hacerlo con cada proyecto nuevo.

    1. Lo primero que tenemos que hacer es abrir una TERMINAL o CONSOLA. Podemos usar el programa "GIT BASH" que se descarga cuando instalamos GIT o usar la terminal de nuestra PC o la terminal que esta incorporada dentro de Visual Studio Code, entre otras.
    2. El primer paso es vincular un correo electrónico. A esto lo hacemos en la terminal con el comando git config --global user.email "tucorreo@electronico.com"
    3. Después debemos dar un nombre de usuario. A esto lo hacemos con el comando git config --global user.name "Nombre_de_usuario_sin_espacios"

    Con esto ya tendríamos configurado un usuario de GIT, nos habilita para crear un repositorio LOCAL. Si quisieramos consultar estos datos podemos usar el comando git config --global -e

  • Creando un usuario en GitHub

    1. Lo primero es ingresar a Registrarse (Sign up) en la página de GitHub. Nos pedirá nuestro correo electrónico, contraseña, un nombre de usuario, preguntará si queremos recibir ofertas o novedades (contestaremos con y o n, "si o no"), y por último, nos pedirá resolver una prueba "captcha" para verificar que somos humanos.
    2. GitHub enviará un mail al correo electrónico que le proporcionaste con un CÓDIGO DE VERIFICACIÓN, que debemos ingresar en este paso.

    Con esto ya tenemos una cuenta en GitHub, y un repositorio REMOTO donde podremos subir y compartir nuestro repositorio local de GIT.

Trabajando con GIT (repositorio LOCAL)

  1. Crear un directorio (una carpeta contenedora del proyecto)
  2. Abrir el directorio con Visual Studio Code (VSC)
  3. Nos dirigiremos a la consola de VSC seleccionando desde el menú principal (o menú hamburguesa) "Terminal > Nueva Terminal". Desde ahí podemos elegir o agregar con el botón "+" algun tipo de terminal. Se recomienda Powershell pero también podemos usar Bash, que es la que entiende exclusivamente los comandos de GIT.
  4. Una vez allí, inicializamos el repositorio con el comando git init, que se usa por unica vez en este proyecto.
  5. Ahora agregamos contenido al directorio (archivos, otras carpetas) en el Editor de VSC
  6. Podemos ver el estado de nuestro repositorio usando un comando: git status. En este punto nos mostrará que nuestro contenido no tiene seguimiento, eso quiere decir que no está en la etapa de staging. Nos damos cuenta porque los nombres de archivos y carpetas se enlistan en rojo.
  7. Para agregar el contenido del directorio a la etapa de staging tenemos que usar el comando git add "nombre_de_tu_archivo.ext" o git add . . El último de estos comandos agrega todos los archivos y carpetas automáticamente. Ahora si volvemos a usar "git status", los archivos y carpetas se enlistarán en verde.
  8. Ahora ya podemos enviar todo el directorio del área de staging al Repositorio LOCAL usando el comando git commit -m "agrega un mensaje descriptivo aca". A ésto se lo conoce como hacer commit.

Trabajando con GitHub (repositorio REMOTO)

Teniendo todo el directorio en el repositorio Local, ya podemos trabajar con la página GitHub, que es un repositorio remoto.

  1. Habiendo ingresado a nuestra cuenta, primero creamos un nuevo repositorio remoto (boton verde) al que le vamos a dar un nombre y presionaremos el boton "crear repositorio" (Create repository).
  2. A él enviaremos desde la Terminal Bash de VSC todo el repositorio LOCAL.

GitHub nos proporcionará todos los comandos necesarios para trabajar en la terminal de VSC.

Trabajando con GIT y GitHub en conjunto

Copiamos uno por uno los comandos proporcionados por git, en el orden que son dados.

  1. Copiamos el comando git branch -M main de GitHub y lo pegamos en la terminal VSC
  2. Ahora colocamos un comando que nos permitirá vincular ambos repositorios git remote add origin https://github.com/TuUsuario/TuRepositorio.git. A ésto lo realizaremos por única vez para este proyecto.
  3. Finalizamos con git push -u origin main, que envía todo lo que está en GIT (R. local) a GitHub (R. remoto). La página de GH nos va a pedir una autenticación en el navegador, damos a aceptar.
